==============================================================================
INTRODUCTION                                            *metermeter*

MeterMeter provides real-time poetic meter annotation in Neovim. It uses a
persistent Python subprocess to analyze lines, showing stress highlighting
and meter labels (e.g. "iP") as virtual text.

Scanning is progressive: visible lines first, nearby prefetch lines next,
then remaining scannable lines across the buffer. Cached results are reused
for unchanged lines.

After an initial full-buffer pass, MeterMeter may automatically run a second
pass using the buffer's dominant meter as context. This keeps ambiguous
lines consistent and avoids reusing cached results across incompatible
contexts.

==============================================================================
METER MODEL                                            *metermeter-model*

MeterMeter gates candidate meters by syllable count. For example, iambic
pentameter is only considered for 10 syllables, or 11 when the last syllable
can be treated as an unstressed feminine ending.

For meters with a permitted length deviation, MeterMeter only allows that
deviation at the line edges; it does not insert/delete in the middle of the
line to force-fit a template.

==============================================================================
REQUIREMENTS                                            *metermeter-requirements*

- Neovim 0.10+
- Python 3.11+
- `prosodic` Python package (installed in the project venv)
- `espeak` (optional, used by prosodic for out-of-vocabulary words)

==============================================================================
INSTALLATION                                            *metermeter-install*

lazy.nvim: >lua
    {
      dir = "~/dev/MeterMeter/nvim/metermeter.nvim",
      ft = "metermeter",
      cmd = { "MeterMeterToggle", "MeterMeterEnable", "MeterMeterRescan" },
      keys = {
        { "<leader>mm", "<Plug>(metermeter-toggle)", desc = "Toggle MeterMeter" },
      },
      config = function()
        require("metermeter").setup()
      end,
    }
<
vim-plug: >vim
    Plug '~/dev/MeterMeter/nvim/metermeter.nvim'
<

CONFIGURATION                                           *metermeter-config*

Configure via `g:metermeter_*` globals (set in your init.vim/init.lua before the
plugin loads). MeterMeter auto-calls `setup()` unless you set
`g:metermeter_disable_auto_setup = 1`. >vim
    let g:metermeter_debounce_ms = 100
    let g:metermeter_ui_meter_hint_abbrev = 1
    let g:metermeter_ui_meter_hint_details = 'deviations'
<

                                                *metermeter.debounce_ms*
debounce_ms ~
    Delay (ms) after edits before rescanning.
    Global: `g:metermeter_debounce_ms`
    Default: `80`

                                                *metermeter.rescan_interval_ms*
rescan_interval_ms ~
    Periodic rescan interval (ms). `0` disables.
    Global: `g:metermeter_rescan_interval_ms`
    Default: `0`

                                                *metermeter.prefetch_lines*
prefetch_lines ~
    Lines scanned around cursor beyond visible range.
    Global: `g:metermeter_prefetch_lines`
    Default: `5`

                                                *metermeter.require_trailing_backslash*
require_trailing_backslash ~
    Only annotate lines ending with `\`.
    Global: `g:metermeter_require_trailing_backslash` (also supports `b:metermeter_require_trailing_backslash`)
    Default: `false`

                                                *metermeter.ui.stress*
ui.stress ~
    Enable stress span highlighting.
    Global: `g:metermeter_ui_stress`
    Default: `true`

                                                *metermeter.ui.meter_hints*
ui.meter_hints ~
    Show meter label at end of line (optionally with notes like "(fem, inv)";
    see |metermeter.ui.meter_hint_details|).
    Global: `g:metermeter_ui_meter_hints`
    Default: `true`

                                                *metermeter.ui.meter_hint_abbrev*
ui.meter_hint_abbrev ~
    Abbreviate meter names in the end-of-line hint (e.g. "iP", "t4").
    Global: `g:metermeter_ui_meter_hint_abbrev`
    Default: `true`

                                                *metermeter.ui.meter_hint_details*
ui.meter_hint_details ~
    Controls how much detail is shown in the end-of-line meter hint.

    Values: `"off"` | `"deviations"` | `"always"`
    Global: `g:metermeter_ui_meter_hint_details`
    - `"off"`: show only the meter name
    - `"deviations"`: show only non-default notes (e.g. feminine endings and
      binary-meter substitutions like inversions/spondees/pyrrhics)
    - `"always"`: always show ending (`masc`/`fem`) plus any substitutions

    Notes:
    - `fem`: feminine ending (extra trailing `U`)
    - `inv`: inverted binary foot (`SU` in iambic, `US` in trochaic)
    - `spon`: spondee (`SS`)
    - `pyrr`: pyrrhic (`UU`)

    Notes are currently emitted for iambic/trochaic meters only.

    Default: `"deviations"`

                                                *metermeter.ui.confident_threshold*
ui.confident_threshold ~
    Confidence >= this uses bright highlight; below uses dim.
    Global: `g:metermeter_ui_confident_threshold`
    Default: `0.7`

                                                *metermeter.ui.loading_indicator*
ui.loading_indicator ~
    Show spinner on lines being analyzed.
    Global: `g:metermeter_ui_loading_indicator`
    Default: `true`

                                                *metermeter.cache.max_entries*
cache.max_entries ~
    Per-buffer LRU cache capacity.
    Global: `g:metermeter_cache_max_entries`
    Default: `5000`

                                                *metermeter.debug_dump_path*
debug_dump_path ~
    Output path for |:MeterMeterDebug|.
    Global: `g:metermeter_debug_dump_path`
    Default: `"/tmp/metermeter_nvim_dump.json"`

COMMANDS                                                *metermeter-commands*

                                                *:MeterMeterEnable*

                                                *:MeterMeterDisable*

==============================================================================
LUA API                                                 *metermeter-api*

All functions are available via `require("metermeter")`.

                                                *metermeter.setup()*
setup({opts})           Initialize the plugin with optional config overrides.
                        Reads configuration from `g:metermeter_*` globals;
                        globals override any values passed in {opts}.
                        Must be called before any other function.

                                                *metermeter.enable()*
enable({bufnr})         Enable MeterMeter for the given buffer.
                        Pass `0` for the current buffer.

                                                *metermeter.disable()*
disable({bufnr})        Disable MeterMeter for the given buffer.

                                                *metermeter.toggle()*
toggle({bufnr})         Toggle MeterMeter for the given buffer.

                                                *metermeter.rescan()*
rescan({bufnr})         Invalidate the cache and re-analyze the buffer.

                                                *metermeter.statusline()*
statusline({bufnr})     Return a string for statusline integration.
                        Shows "MM: iambic pentameter" when active,
                        "MM: ..." while scanning, or error info on failure.

STATUSLINE                                              *metermeter-statusline*

MeterMeter injects into the statusline automatically. If lualine is
installed, it adds a component to `lualine_x`. Otherwise, it patches the
statusline of each MeterMeter window directly.

For manual integration: >lua
    require("lualine").setup({
      sections = {
        lualine_x = { function() return require("metermeter").statusline() end },
      },
    })
<

HEALTH CHECK                                            *metermeter-health*

Run `:checkhealth metermeter` to verify your setup. It checks:
- Neovim version (>= 0.10)
- Python version (>= 3.11)
- `prosodic` importability
- `espeak` availability
- Subprocess status
- Whether `setup()` has been called

==============================================================================
TROUBLESHOOTING                                         *metermeter-troubleshooting*

No annotations appear ~
    Check that `&filetype` includes "metermeter" and that the statusline
    shows "MM: ..." or a meter name. Use `:MeterMeterToggle` to force-enable
    on any buffer.

Meter labels change after opening a file ~
    MeterMeter may do a second pass once it detects a strong dominant meter.
    This is expected; use `:MeterMeterRescan` if the buffer looks stale.

Unexpected highlights ~
    Run `:MeterMeterDebug` and inspect the JSON at |metermeter.debug_dump_path|.

espeak warnings ~
    Install espeak: `brew install espeak` (macOS) / `apt install espeak`
    (Linux). Without it, OOV words use a neural heuristic which may be
    slower or less accurate.

Subprocess crashes ~
    Check `:MeterMeterDebug` for error details. The subprocess restarts
    automatically up to 3 times per minute.
